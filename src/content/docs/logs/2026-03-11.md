## 一、部署方案全景图

目前主流的几家都推出了"一键部署"：

| 厂商 | 方案 | 门槛 | 适合谁 |
|------|------|------|--------|
| **阿里云** | 计算巢预装镜像 | 低 | 小白首选 |

| **Kimi** | KimiClaw托管 | 低 | 长文需求 |

### 阿里云 vs 腾讯云对比

**阿里云计算巢**
- ✅ 官方预装镜像，5分钟开箱
- ✅ 可视化配置API Key
- ✅ 香港节点免备案
- ❌ 价格稍贵（¥99/年起）

**腾讯云轻量**
- ✅ 学生价¥10/月（真香）
- ✅ 秒杀活动¥10-20/年
- ❌ 需手动命令行安装
- ❌ 没有官方镜像支持

**结论**：
- 能抢到腾讯云秒杀 → 腾讯云
- 求稳定省心 → 阿里云
- 纯小白 → 阿里云

## 二、API计费的水有多深

### 你以为的免费 vs 实际

很多教程说"免费部署"，但**部署免费 ≠ 使用免费**！

**真实成本构成**：
1. **服务器费用**（必选）：¥10-200/月
2. **API调用费用**（隐藏）：按token计费
3. **流量费用**（可能超）：¥0.8/GB

### Token计费详解

**什么是Token？**
简单说就是"词块"，中英文混合 roughly 1 token ≈ 0.75个汉字。

**百炼价格表**：
| 模型 | 输入 | 输出 |
|------|------|------|
| qwen-turbo | ¥0.004/千token | ¥0.008/千token |
| qwen-plus | ¥0.004/千token | ¥0.012/千token |
| qwen-max | ¥0.04/千token | ¥0.12/千token |

**实际消耗示例**：
- 简单问答（500 token）：¥0.003-0.006
- 长文分析（3000 token）：¥0.02-0.05
- 代码生成（5000 token）：¥0.03-0.08

看起来不多？但累积起来：
- 每天50次对话：¥2-5/天 = ¥60-150/月
- 每天200次：¥8-20/天 = ¥240-600/月

### 免费额度的真相

- **阿里云百炼**：新用户送100万-1000万token，有效期30-90天
- **DeepSeek**：送500万token，30天有效
- **Kimi**：充值赠送，不是纯免费

**关键问题**：
- 免费额度用完自动扣费（可能欠费！）
- 没有提醒的话，账单可能很吓人
- 不同模型价格差10倍（turbo vs max）

## 三、Ollama本地部署真的是零成本吗？

### 原理

Ollama把开源模型（Llama、Qwen等）下载到本地，用你自己的CPU/GPU跑，不调用云端API。

```
你的问题 → 本地电脑计算 → 本地生成回答
     ↑                              ↓
   你的硬件  <────────────────  不上云
```

### 隐性成本

| 成本类型 | 说明 | 估算 |
|----------|------|------|
| **硬件要求** | 需要较好的CPU/GPU | Mac M系列可以，Intel核显卡 |
| **电费** | 满载50-200W | ¥0.5-3/天 |
| **存储** | 7B模型4GB，70B模型40GB+ | 硬盘空间 |
| **时间** | 配置麻烦 | 新手2-4小时 |
| **性能** | 本地小模型 < 云端大模型 | 质量差距明显 |

### 适合谁

✅ 技术极客、隐私敏感、无网环境、预算极低
❌ 小白用户、求高质量回答、不想折腾

## 四、今天的实战：修复博客部署

昨晚遇到GitHub Pages不自动构建Astro的问题，今天解决思路：

**问题**：GitHub Pages默认不会`npm install`和`npm run build`，直接显示源码。

**解决方案对比**：
1. **GitHub Actions**（推荐）：自动构建，但token需要workflow权限
2. **本地构建+推dist**：手动构建后只推送产物，麻烦但可控
3. **改用Cloudflare Pages**：自动构建，更简单

目前暂时用方案2应急，后续考虑迁移到Cloudflare Pages。

## 五、避坑指南

### 1. "永久免费"的套路
很多教程说"永久免费部署OpenClaw"，实际上：
- 服务器要么限时免费，要么根本不存在
- API调用永远是按量计费（除非本地Ollama）
- 免费额度用完必收费

### 2. 忘记关服务的后果
7×24小时运行，如果被公开访问，可能被别人刷API：
- 设置dmPolicy=pairing（需验证才能用）
- 设置allowFrom白名单
- 定期检查账单

### 3. 模型选择陷阱
默认可能用qwen-max（最贵），手动改成turbo或plus：
```bash
openclaw config set model.modelId qwen-turbo
```

### 4. 上下文累积
长对话历史会累积token，越来越贵：
- 定期清空对话
- 限制上下文长度
- 重要信息写入MEMORY.md而不是带着上下文
